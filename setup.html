This hub reads an Anki collection and personalizes its pages around it. The features below are modular — pick whichever fit how you study. Anki setup is the one shared foundation; video mining and manga mining are optional add-ons that feed new cards into the same deck. Pick a phone platform to tailor the app recommendations. Install Anki + AnkiConnect

the foundation — everything else plugs into this
1

Install Anki (desktop)

Download from apps.ankiweb.net and create a free AnkiWeb account — that account is what syncs the deck to a phone.

2

Add the AnkiConnect add-on

In Anki: Tools → Add-ons → Get Add-ons, paste code 2055492159, then restart Anki. AnkiConnect opens a small local API on 127.0.0.1:8765 that this hub (and the mining tools) read from.

Recommended decks

the fastest route to “mining is feasible”

Mining works best once you can read kana instantly and recognize ~1,000–1,500 common words — below that, every sentence has too many unknowns to pick just one. The community-standard path there: kana first, then one starter vocab deck to completion, learning kanji through the words rather than in isolation. Download a deck (.apkg) and load it with File → Import in Anki; everything imported shows up in this hub on the next sync.

Step 0 · Kana

Japanese Kana (Mnemonics, 2× Audio, Stroke)

All hiragana + katakana with Tofugu-style picture mnemonics, male and female audio, and stroke diagrams. Do hiragana before katakana. A week or two of this (plus the kana drills here) and reading kana stops being the bottleneck. Tofugu also publishes a standalone katakana deck.

Step 1 · Vocabulary — the main event

Kaishi 1.5k

The current community-standard starter deck: 1,500 high-frequency words with native audio, example sentences, furigana, and pitch-accent graphs. It replaced Core 2000 as the default recommendation because the sentences and audio are simply better. Finishing this deck ≈ the point where i+1 mining becomes comfortable. Modular and open source on GitHub.

Optional · Kanji help

RRTK 450

The 450 most common kanji as recognition cards with keywords, images, and sample vocab. Modern consensus is that kanji are best learned through words (Kaishi already does this), so treat this as a booster only if characters still feel like indistinguishable noise a few hundred words in — it makes new vocab stick without the months a full RTK run costs. Pair it with the radicals page here for the building blocks.

Alternatives

Tango N5/N4 · Core 2k/6k

Tango N5 → N4 teaches via full JLPT-aligned sentences (companion decks to a paid book series) — good if you prefer sentence cards. Core 2000/6000 is the older standard Kaishi replaced — still solid, just dated audio and sentences. Pick one starter deck and finish it; running two vocab decks in parallel doubles reviews, not progress.

After the starter deck, stop adding pre-made decks — from there, mining your own cards (Video / Manga tabs above) beats any shared deck, because every card comes from something you actually watched or read.

Sources: these picks reflect the consensus across Migaku’s 2026 deck roundup, the Donkuri learning guide (by Kaishi’s authors), Animecards’ guide, Tatsumoto on kana decks, learnkanji.guide, and Cade McNiven’s kanji-through-vocab writeups.

Connect this hub to Anki

the exact sequence that works
1

Sync — and watch the Anki window

With Anki open, click Sync from Anki on the home page. The very first sync makes AnkiConnect show an “Allow connection?” popup — inside the Anki window, usually hidden behind the browser. Bring Anki to the front and click Yes, then sync again.

2

If the hub still can’t find AnkiConnect: the CORS fix

A hub opened by double-clicking the file has the origin null, which AnkiConnect blocks by default. Open Tools → Add-ons → AnkiConnect → Config and make sure it reads:

{
  "webBindAddress": "127.0.0.1",
  "webBindPort": "8765",
  "webCorsOriginList": ["http://localhost", "null"]
}

Restart Anki (the config only reloads on restart), hard-reload the hub (Ctrl/Cmd+Shift+R), and sync again. This is the fix that resolves nearly every “can’t find it” case.

Using a hosted copy of the hub instead of a local file? Add that site’s origin to the same list — e.g. for the GitHub Pages copy: "https://rory-97.github.io". Origins are scheme + domain only, no path.

3

Sanity check

With Anki open, visit http://127.0.0.1:8765 in the browser — it should say “AnkiConnect v.6”. If it doesn’t, AnkiConnect isn’t running and no page can reach it.

4

Optional: serve over localhost

Running the hub from http://localhost instead of a double-clicked file sidesteps every file:// quirk: from the hub folder run python -m http.server 8000 (or npx serve), open http://localhost:8000, and add "http://localhost:8000" to webCorsOriginList.

After a successful sync the hub tags every card as known (mature — interval of 21+ days) or learning, and every page that adapts to the deck updates automatically. Re-sync any time. The hub also accepts a plain Anki export (File → Export → Notes in Plain Text) via the Connect Anki pill — no live sync required at all.

携

Review on your phone

outside the hub — Anki syncs it back
1

AnkiDroid free

Install AnkiDroid from the Play Store and log into the same AnkiWeb account — the whole deck syncs down. Do daily reviews anywhere; the next AnkiWeb sync carries the progress back to desktop, and the hub picks it up on its next Sync from Anki.

1

AnkiMobile one-time paid

Install AnkiMobile from the App Store (a one-time purchase that funds Anki’s development) and log into the same AnkiWeb account — the whole deck syncs down. Do daily reviews anywhere; the next AnkiWeb sync carries the progress back to desktop, and the hub picks it up on its next Sync from Anki.

The hub itself is desktop-only — phone reviews happen in the Anki app, and AnkiWeb sync is the bridge that keeps everything in one collection.

映

Mine vocabulary from video

Crunchyroll & YouTube → Anki · optional add-on

Turn lines from streaming video into Anki cards in two keystrokes. Requires the Anki setup tab first — asbplayer writes cards through the same AnkiConnect.

1

Install asbplayer + Yomitan

Add asbplayer and Yomitan to the browser (Chrome/Edge). In Yomitan, import a dictionary (JMdict is the standard choice). In asbplayer settings, point it at the target Anki deck and note type.

2

Crunchyroll: bring Japanese subtitles

Crunchyroll only ships translated subtitles, so mining needs a matching Japanese .srt/.ass file from jimaku.cc. Play the episode, drag the subtitle file onto the video, and asbplayer overlays it. If timing drifts, nudge it with A/D.

3

YouTube: usually automatic

Videos with Japanese captions work directly. For videos without captions, drag in a subtitle file the same way.

4

Make the card

When a worthwhile line appears, press Ctrl + Shift + X — asbplayer captures the sentence, audio, and a screenshot. Yomitan fills in the word, reading, and meaning. Export, and the card is in Anki; the next hub sync picks it up.

🎯 The sweet spot is “i+1” sentences — every word known except one. A handful of good cards per episode beats dozens of half-understood ones.

On Android: jidoujisho can watch and mine video on the phone itself, exporting cards to AnkiDroid. Desktop mining is still the smoother workflow.

On iPhone: there’s no solid on-device video-mining tool — mine on desktop and use AnkiMobile for reviews; AnkiWeb sync keeps the deck identical everywhere.

漫

Mine vocabulary from manga

OCR turns pages into text · optional add-on

Manga text is baked into the images, so an OCR step replaces asbplayer — the dictionary and Anki half of the workflow stays identical. This only works with Japanese-language editions (raws), not translated releases. Requires the Anki setup tab first.

1

Desktop: mokuro

Run mokuro once over a volume — it OCRs every page into a .mokuro file. Open that in the mokuro web reader: the text in the panels becomes selectable, Yomitan pops up definitions on hover, and mining to Anki works exactly like video.

2

Android: Chimahon

Chimahon is a Mihon/Komikku-lineage manga reader with built-in dictionary lookup, mokuro support, and one-tap AnkiDroid card creation — reading and mining in one app. (Plain Komikku has no dictionary/OCR/mining; Chimahon is the same reading experience with those added.)

2

iPhone: read there, mine on desktop

iOS has no one-tap equivalent of Chimahon. The mokuro web reader works in Safari for reading OCR’d volumes, and Manabi Reader is a native iOS reading app with dictionary lookups and Anki export for web/text content. For manga specifically, the reliable flow is: read anywhere, mine on desktop.

The daily loop

how the pieces fit together

見る・読むWatch (asbplayer + Japanese subs) or read (mokuro / a manga reader) native material.
採るMine a handful of i+1 sentences into Anki.
覚えるReview the deck daily — the Anki phone app syncs it all back via AnkiWeb.
試すSync the hub; it adapts its kanji, quizzes, and stats to the deck.

さあ、始めよう！  Set up Anki + AnkiConnect once, add whichever mining tools fit, and the rest is watching and reading things worth enjoying — the hub turns that into review.
